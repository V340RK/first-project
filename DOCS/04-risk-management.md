## Частина 1: Розмір позиції

### Правило 1% (базове правило бота)

```
Розмір позиції = (Депозит × Ризик%) / Відстань до стопу

Приклад:
  Депозит:        $10,000
  Ризик на угоду: 1% = $100
  Ціна входу:     $50,000
  Стоп-лос:       $49,500  (відстань = $500 або 1%)

  Розмір позиції = $100 / $500 = 0.2 BTC
```

Таким чином, якщо стоп спрацює — ти втрачаєш рівно $100 (1% депозиту), незалежно від ціни монети.

### Таблиця допустимого ризику

| Ситуація | Ризик на угоду |
|---|---|
| Нормальна робота | 1.0% |
| Після 2 збиткових угод підряд | 0.5% |
| Після 4 збиткових угод підряд | 0.25% |
| Стоп торгівлі (дивись Circuit Breaker) | 0% |

## Частина 2: Стоп-лос

### Принцип розміщення стопу

Стоп **завжди** ставиться за технічним рівнем, а НЕ за фіксованою відстанню.

```
Для LONG позиції:
  Стоп = нижче зони Order Block / нижче зони FVG / нижче absorption zone
  + 1-2 тіки буфер (щоб не зачепити випадковий шпиль)

Для SHORT позиції:
  Стоп = вище зони Order Block / вище зони FVG / вище absorption zone
  + 1-2 тіки буфер
```

### Методи стопу

| Метод | Коли використовувати |
|---|---|
| **Structure Stop** | Основний. Стоп за SMC-рівнем (OB, FVG) |
| **ATR Stop** | Стоп = Ціна ± (ATR × 1.5). Адаптується до волатильності |
| **Fixed Stop** | Тільки для швидкого тестування. В реальній торгівлі не використовувати |

### ATR (Average True Range) — що це?
Середній розмір руху ціни за N свічок. Якщо ATR за 14 свічок = $200, значить ціна в середньому рухається на $200 за свічку. Стоп за ATR = $200 × 1.5 = $300.

```python
# Розрахунок ATR
def atr(candles, period=14):
    true_ranges = []
    for i in range(1, len(candles)):
        tr = max(
            candles[i].high - candles[i].low,
            abs(candles[i].high - candles[i-1].close),
            abs(candles[i].low - candles[i-1].close)
        )
        true_ranges.append(tr)
    return sum(true_ranges[-period:]) / period
```

## Частина 3: Тейк-профіт

### Мінімальне співвідношення ризик/прибуток

Бот **ніколи** не входить в угоду якщо потенційний прибуток менший ніж подвійний ризик.

```
Мінімальний R:R = 1:2

Якщо Стоп = $100 збитку → мінімальний Тейк = $200 прибутку

Чому 1:2?
  При 1:2 достатньо перемагати лише в 34% угод щоб бути в плюсі:
  3 перемоги × $200 = $600
  6 поразок × $100 = -$600
  Але реально якщо сигнал хороший — win rate вище 40%, тобто ми в плюсі
```

### Методи виходу

**Варіант A: Фіксований тейк**
```
Тейк = Ціна входу ± (Відстань стопу × R:R мультиплікатор)

Приклад Long:
  Вхід: $50,000
  Стоп: $49,500 (відстань $500)
  R:R = 2
  Тейк = $50,000 + ($500 × 2) = $51,000
```

**Варіант B: Частковий вихід (рекомендований)**
```
50% позиції закрити на 1R (перший тейк)
50% позиції тримати з трейлінг-стопом

Переваги:
  - Фіксуємо гарантований прибуток рано
  - Але не закриваємо все якщо рух продовжується
```

**Варіант C: SMC-цілі**
```
Тейк = наступний ключовий рівень:
  - Наступний Order Block / FVG зона
  - Previous Day High/Low
  - Round number ($50,000, $51,000)
```

## Частина 4: Circuit Breaker (Автоматичний вимикач)

Це система яка **автоматично зупиняє** торгівлю при певних умовах.

### Рівні:

```
РІВЕНЬ 1 — ОБЕРЕЖНІСТЬ:
  Умова: денний збиток > 2% депозиту
  Дія: Зменшити розмір позиції вдвічі (з 1% до 0.5%)

РІВЕНЬ 2 — ПАУЗА:
  Умова: денний збиток > 5% АБО 4 збиткових угоди підряд
  Дія: Зупинити нові угоди на ПОТОЧНИЙ ДЕНЬ
        Всі відкриті позиції закрити

РІВЕНЬ 3 — ЗУПИНКА ТИЖНЯ:
  Умова: тижневий збиток > 10%
  Дія: Зупинити торгівлю на весь тиждень
        Потребує ручного перезапуску

РІВЕНЬ 4 — KILL SWITCH (аварійна зупинка):
  Умова: збиток депозиту > 20% АБО помилка системи
  Дія: 
    1. Скасувати ВСІ відкриті ордери
    2. Закрити ВСІ позиції
    3. Відправити алерт (Telegram/Email)
    4. Заблокувати подальшу роботу до ручного відновлення
    5. Записати лог події
```

### Kill Switch — обов'язкові вимоги:
- Незалежний від основної логіки (окремий процес/потік)
- Спрацьовує навіть якщо основний код впав з помилкою
- Не можна скасувати автоматично — тільки людина

## Частина 5: Очікувана цінність (EV)

Перед запуском будь-якої нової стратегії — рахуємо чи вона математично прибуткова.

```
EV = (Win Rate × Середній прибуток) - (Loss Rate × Середній збиток)

Приклад:
  Win Rate = 45% (45 виграшних з 100)
  Loss Rate = 55%
  Середній прибуток = $200 (R:R 2:1 при стопі $100)
  Середній збиток = $100

  EV = (0.45 × $200) - (0.55 × $100)
     = $90 - $55
     = +$35 за угоду ✓ (позитивна)

Мінімальна вимога: EV > 0 після врахування комісій біржі
```

## Частина 6: Слипедж і комісії

Скальпінг дуже чутливий до витрат. Потрібно враховувати:

```
Комісія Binance Futures (маркет-ордер): 0.04% на вхід + 0.04% на вихід = 0.08% на угоду
Слипедж (ковзання ціни): 0.01-0.05% залежно від ліквідності

Загальні витрати на угоду: приблизно 0.1%

Якщо ціль = 0.3% прибутку → реальний прибуток = 0.2% після витрат

ВАЖЛИВО: Якщо ціль менша за 0.15% — угода невигідна після витрат!
```
