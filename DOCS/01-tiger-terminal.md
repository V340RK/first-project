# Tiger.Trade — Термінал

## Що таке Tiger.Trade?

Tiger.Trade — це **професійний торговий термінал** (Windows/macOS), створений для трейдерів, які торгують на основі потоку ордерів (order flow).

Простими словами: це програма, яка показує не просто ціну і свічки, а **що саме відбувається всередині ринку** — скільки покупців, скільки продавців, де стоять великі ордери, хто агресивно купує або продає прямо зараз.

---

## З якими біржами працює?

Binance, ByBit, BitMEX, Bitfinex, OKX, MEXC, Gate.io, Bitget, Deribit та інші — через агрегатор TigerX.

---

## Що термінал вміє показувати?

### 1. Footprint / Cluster Chart (Кластерний графік)
Це основний інструмент. Замість звичайної свічки — таблиця, де кожен рядок = один ціновий рівень, і в кожній клітинці вказано:
- скільки угод пройшло по ціні покупки (Bid)
- скільки — по ціні продажу (Ask)

Є 30+ режимів відображення: Volume, Delta, Bid×Ask, Imbalance та інші.

### 2. Dynamic DOM (Динамічний стакан)
Показує не просто поточний стан стакану, а **як він змінюється** в реальному часі. Видно, коли великі ордери з'являються або зникають.

### 3. Trade Feed / Tape (Стрічка угод)
Список кожної виконаної угоди: час, ціна, об'єм, напрямок (покупець чи продавець був агресором). Можна фільтрувати великі принти (великі угоди).

### 4. Delta
Різниця між об'ємом покупок і продажів. Якщо delta сильно позитивна — покупці агресивні. Якщо негативна — продавці.

### 5. Volume Profile
Горизонтальна гістограма де показано скільки об'єму торгувалось на кожному ціновому рівні за сесію.

## Як бот підключається до терміналу?

Tiger.Trade має **вбудований WebSocket-сервер** на порту `7819`.

### Що цей API вміє:
- Перемикати символ (монету) у вказаній групі графіків
- Синхронізувати символ між зовнішнім скринером і терміналом

### Що цей API НЕ вміє:
- Передавати дані order flow назовні (footprint, delta, DOM)
- Відправляти торгові накази (buy/sell)

```
Простими словами: через WebSocket API ми можемо тільки
"сказати терміналу показати монету X", але не отримати
дані з терміналу в бота.
```

---

## Як тоді отримати дані для бота?

Є два шляхи:

### Шлях A: Плагін всередині терміналу (C#)
Пишемо код на C# — він завантажується прямо в Tiger.Trade як власний індикатор.  
Плюс: повний доступ до footprint, delta, DOM всередині терміналу.  
Мінус: складніше розробляти, тільки Windows.

```
Приклад: TTPlugins (відкритий код на GitHub: TTPlugins/TTPlugins)
— це набір готових плагінів для Tiger.Trade написаних саме так.
```

### Шлях B: Пряме підключення до біржі (Python)
Бот підключається **напряму до Binance/ByBit** через їх WebSocket API і отримує:
- стакан ордерів (order book) в реальному часі
- потік угод (trade stream)
- агреговані угоди

Плюс: простіше, не залежить від терміналу, можна на будь-якій ОС.  
Мінус: немає готового footprint — треба рахувати самому.

## Наш підхід

Ми будуємо бота за **Шляхом B** — пряме підключення до біржі через Python.  
Tiger.Trade використовуємо як **візуальний інструмент** для ручного аналізу і контролю.

```
Бот (Python) ←→ Binance/ByBit WebSocket API
Tiger.Trade ←→ Binance/ByBit (для відображення)
```

Це дає нам:
- Повний контроль над логікою
- Незалежність від терміналу
- Можливість тестувати і запускати без GUI
