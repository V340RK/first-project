# Scalping Bot — Документація

## Що це за проект?

Автоматизований скальпінг-бот для торгівлі криптовалютою через термінал **Tiger.Trade**.  
Бот читає дані з ринкового стакану (DOM) і потоку угод (tape), виявляє конкретні мікроструктурні ситуації і відкриває угоди зі строгим ризик-менеджментом.

## Основна ідея роботи бота

```
Ринок генерує дані
        ↓
Бот читає потік угод і DOM в реальному часі
        ↓
Виявляє одну з сигнальних ситуацій
(imbalance / absorption / delta spike / micro pullback / SMC-рівень)
        ↓
Перевіряє ризик-менеджмент (чи дозволено відкривати угоду?)
        ↓
Відкриває угоду з чітким стоп-лосом і тейк-профітом
        ↓
Моніторить угоду → закриває за планом або по стопу
```

## Технологічний стек

| Компонент | Технологія |
|---|---|
| Термінал | Tiger.Trade (Windows) |
| Мова для плагінів всередині терміналу | C# (.NET) |
| Зовнішній рушій сигналів | Python 3.11+ |
| Підключення до бірж (напряму) | REST + WebSocket API (Binance, ByBit, OKX) |
| Комунікація термінал ↔ зовнішній бот | WebSocket (порт 7819) — UI-команди |

## Важливе застереження

> Скальпінг — це торгівля з дуже коротким часом утримання позиції (секунди або хвилини).  
> Більшість прибутку або збитку визначається **не** тим, коли увійшов, а **якою є якість управління ризиком**.  
> Цей бот побудований навколо ризик-менеджменту як пріоритету №1.
